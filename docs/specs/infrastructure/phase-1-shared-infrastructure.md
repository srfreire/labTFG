# Phase 1: Shared Infrastructure Layer

> Status: current | Created: 2026-04-11 | Last updated: 2026-04-11
> References: [general.md](general.md) | [phases.md](phases.md)

## Objective

Build the foundation that both phases depend on: MinIO + Postgres in docker-compose, a `StorageService` wrapping `aioboto3`, a `DatabaseService` wrapping async SQLAlchemy + asyncpg, Alembic migrations, and a unified `shared.init()` lifecycle.

## Requirements

### R1: Docker-compose with MinIO + Postgres

Docker-compose file at repo root with:
- **MinIO** container — ports 9000 (API) + 9001 (console), persistent volume, auto-creates `labtfg` bucket on startup
- **Postgres** container — port 5432, persistent volume, creates `labtfg` database
- Environment variables for credentials with sensible dev defaults
- `.env.example` documenting all required vars

### R2: Settings module

`shared/shared/settings.py` — reads env vars, provides a `Settings` dataclass:
- `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`, `MINIO_BUCKET`
- `POSTGRES_DSN` (full connection string)
- Defaults suitable for local docker-compose (e.g. `localhost:9000`, `postgresql+asyncpg://...`)
- No config files, no YAML — env vars only

### R3: StorageService

`shared/shared/storage.py` — async MinIO/S3 client via `aioboto3`:
- `put(key, data, content_type) -> str` — upload bytes, return key
- `get(key) -> bytes` — download object
- `get_text(key) -> str` — download as UTF-8 string (convenience)
- `put_text(key, text, content_type="text/plain") -> str` — upload string (convenience)
- `list(prefix) -> list[str]` — list object keys under prefix
- `delete(key)` — remove object
- `exists(key) -> bool` — check existence
- Single `aioboto3.Session` reused across calls
- Bucket name from settings, endpoint from settings

### R4: SQLAlchemy async models

`shared/shared/models.py` — SQLAlchemy 2.0 declarative models:

**`Run`** table (new):
- `id` UUID PK (server default)
- `created_at` timestamp
- `problem_description` text
- `status` string
- `s3_report_key` string nullable
- `s3_prefix` string

**`Model`** table (replaces SQLite `models`):
- `formulation_id` string PK
- `class_name` string
- `paradigm` string nullable
- `description` text nullable
- `run_id` UUID FK → runs.id, nullable
- `s3_model_key` string
- `s3_test_key` string nullable
- `registered_at` timestamp
- `metadata` JSONB nullable

**`Experiment`** table (replaces SQLite `experiments`):
- `id` UUID PK
- `created_at`, `updated_at` timestamps
- `description` text
- `status` string (created/simulated/tracked/analyzed/reported)
- `spec` JSONB nullable
- `models_used` JSONB nullable
- `steps` integer nullable
- `seed` integer nullable
- `s3_events_key`, `s3_replay_key`, `s3_tracker_key`, `s3_analyst_key` string nullable
- `s3_pdf_key`, `s3_tex_key` string nullable
- `s3_charts_prefix` string nullable

**`Artifact`** table (new):
- `id` UUID PK
- `s3_key` string unique
- `artifact_type` string (enum values: report, deep_report, formulation, reasoner_spec, model, test, pdf, tex, chart, events, replay, tracker, analyst)
- `run_id` UUID FK nullable
- `experiment_id` UUID FK nullable
- `created_at` timestamp
- `size_bytes` integer
- `content_type` string

### R5: DatabaseService

`shared/shared/database.py` — async Postgres session management:
- `async_engine` from `create_async_engine(settings.POSTGRES_DSN)`
- `async_sessionmaker` for request-scoped sessions
- `get_session()` async context manager yielding `AsyncSession`
- Connection pool settings (pool_size, max_overflow)

### R6: Alembic setup + initial migration

- Alembic configured in `shared/` with async support (`asyncpg`)
- `alembic.ini` + `migrations/` directory inside `shared/`
- Initial migration autogenerated from R4 models
- `env.py` configured to read `POSTGRES_DSN` from env

### R7: Shared lifecycle

`shared/shared/__init__.py` updated:
- `async init(settings=None)` — creates engine, creates MinIO session, ensures bucket exists, runs pending Alembic migrations (or just verifies connectivity)
- `async shutdown()` — closes engine, closes MinIO session
- Exposes `storage: StorageService` and `db: DatabaseService` module-level after init

### R8: Backward-compatible store API

Keep `shared/shared/store.py` working during transition:
- Existing sync functions (`register_model`, `create_experiment`, etc.) continue to work unchanged
- Phase 2 and Phase 1 can migrate incrementally in later phases
- Mark all sync functions as deprecated (simple comment, no warnings)

## Acceptance Criteria

## Technical Notes

- `shared/` is a standalone Python package built with hatchling, referenced as local path dependency by both phases
- Current `store.py` (166 lines) uses raw `sqlite3` — keep it untouched, new code goes in new files
- Both phases already declare `shared` as a dependency in their `pyproject.toml`
- Add `aioboto3`, `sqlalchemy[asyncio]`, `asyncpg`, `alembic` to `shared/pyproject.toml`

## Decisions

| Decision | Rationale |
|----------|-----------|
| `aioboto3` over sync `minio` SDK | Natively async, S3-compatible, no `to_thread` wrapping |
| Alembic inside `shared/` (not a separate package) | Migrations belong with the models they describe |
| Keep `store.py` untouched | Phases 2-3 migrate consumers incrementally; no big-bang cutover |
| Module-level `storage`/`db` singletons after `init()` | Simple access pattern, matches current `_conn` singleton style |
