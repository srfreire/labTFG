\chapter{Conclusiones y posibles ampliaciones}

\section{Balance general}

El trabajo muestra que es viable construir una primera fase de laboratorio
virtual capaz de transformar una descripción en lenguaje natural en modelos de
decisión ejecutables. La aportación no es solo generar código: el sistema
investiga paradigmas, formaliza mecanismos, los adapta a un entorno, genera
modelos Python y conserva evidencia de cada decisión.

La solución combina 3 ideas. Primero, una arquitectura multiagente con
responsabilidades explícitas. Segundo, revisión humana en los puntos de riesgo
científico o de diseño. Tercero, memoria persistente para convertir artefactos
aceptados en conocimiento reutilizable.

\section{Cumplimiento de objetivos}

Los objetivos principales se cumplen. El sistema parte de un problema abierto,
descubre paradigmas, genera formulaciones, produce specs adaptadas y construye
modelos que cumplen el contrato \texttt{DecisionModel}. Además, conserva
artefactos, estado y trazas, por lo que una ejecución puede auditarse sin
depender del contexto conversacional del modelo.

También se cumple el objetivo de mantener al humano dentro del ciclo. El
usuario puede aprobar paradigmas, seleccionar formulaciones, revisar specs,
rechazar modelos y forzar nuevas ejecuciones. Esta estructura reduce el riesgo
de que un error temprano llegue al código final sin ser detectado.

\section{Conclusiones técnicas}

La primera conclusión técnica es que los agentes de lenguaje funcionan mejor
cuando no operan como una entidad monolítica. Separar investigación,
formalización, adaptación y construcción ayuda a localizar fallos y reduce la
presión sobre la ventana de contexto.

La segunda conclusión es que la memoria debe formar parte del diseño desde el
principio. La combinación de PostgreSQL, MinIO, Neo4j y Qdrant añade
complejidad, pero permite conservar procedencia, hechos, relaciones y artefactos
completos. Sin esa capa, el sistema sería difícil de auditar, evaluar o
reanudar.

La tercera conclusión es que los modelos actuales aceleran implementación,
pruebas y refactorización, pero no eliminan el criterio humano. En este proyecto
la intervención humana sigue siendo necesaria para validar interpretaciones
científicas y aceptar modelos generados \cite{chen2021codex,yang2024sweagent}.

\section{Cierre}

El proyecto demuestra que un sistema multiagente con herramientas, memoria,
artefactos persistentes y revisión humana puede abordar una tarea que no se
reduce a generación de texto ni a generación de código. La contribución central
es el flujo trazable que conecta conocimiento científico, formulación,
adaptación, implementación y prueba dentro de un laboratorio virtual
\cite{yao2023react,lewis2020rag,packer2023memgpt,chen2021codex}.
